# Admin 登录页 + 管理后台首页 (pro-components)

## Context

项目是 Rails 8.1 + Inertia.js + React (Vite) 的新项目，前端 antd v6 + pro-components 已安装但未使用，pages/ 目录为空，无认证实现，无数据库迁移。需要从零搭建：用户认证（手写 session）+ 登录页 + 带 ProLayout 的 admin dashboard。

## 1. 后端：认证基础

### 1a. 启用 bcrypt
- **`Gemfile`** — 取消注释 `gem "bcrypt", "~> 3.1.7"`
- 运行 `bundle install`

### 1b. User 模型
- 运行 `bin/rails generate model User email:string:uniq password_digest:string name:string` 
- 运行 `bin/rails db:migrate`
- **`app/models/user.rb`** — 添加 `has_secure_password`

### 1c. Seed 初始管理员
- **`db/seeds.rb`** — 创建默认 admin 用户（admin@example.com / password123）
- 运行 `bin/rails db:seed`

### 1d. Sessions Controller
- **`app/controllers/admin/sessions_controller.rb`** — 新建
  - `new` → 渲染 Inertia 登录页
  - `create` → 验证 email/password，session 存 user_id，redirect_to admin_root
  - `destroy` → 清除 session，redirect_to login

### 1e. Admin Base Controller 加认证
- **`app/controllers/admin_controller.rb`** — 添加 `before_action :require_login` 和 `current_user` 辅助方法
- **`app/controllers/inertia_controller.rb`** — 取消注释 `inertia_share user:` 共享当前用户信息

### 1f. 路由
- **`config/routes.rb`** — 添加 admin 命名空间路由：
  ```ruby
  namespace :admin do
    root "dashboard#index"
    get  "login", to: "sessions#new"
    post "login", to: "sessions#create"
    delete "logout", to: "sessions#destroy"
  end
  ```

### 1g. Admin Dashboard Controller
- **`app/controllers/admin/dashboard_controller.rb`** — 新建，`index` action 渲染 Inertia 页面

## 2. 前端：TypeScript 路径别名修复

- **`tsconfig.app.json`** — 将 `@/*` 和 `~/*` 的路径映射从 `app/javascript/*` 修正为 `app/frontend/*`

## 3. 前端：Admin 布局 (ProLayout)

### 3a. Admin 布局组件
- **`app/frontend/layouts/AdminLayout.tsx`** — 新建
  - 使用 `@ant-design/pro-components` 的 `ProLayout` 组件
  - 侧边栏菜单配置（Dashboard、Users 等占位项）
  - 顶栏显示当前用户名 + 退出按钮
  - 使用 Inertia 的 `router.post` 调用 logout

## 4. 前端：登录页

### 4a. 登录页组件
- **`app/frontend/pages/admin/Login.tsx`** — 新建
  - 使用 `@ant-design/pro-components` 的 `LoginForm`（或 `ProForm` + `LoginForm.Item`）
  - Ant Design 主题：使用 `ConfigProvider` + `theme.algorithm` 配置品牌色
  - email + password 表单字段
  - 使用 Inertia `router.post` 提交到 `/admin/login`
  - 错误提示（从 Inertia shared props 获取 flash errors）

### 4b. Inertia 入口适配
- **`app/frontend/entrypoints/inertia.tsx`** — 保持现有 `createInertiaApp` 不变（页面解析器已指向 `../pages`）

## 6. 验证步骤

1. `bundle install` — bcrypt 安装成功
2. `bin/rails db:create db:migrate db:seed` — 数据库就绪，admin 用户创建
3. `bin/rails dev` — 启动 Rails 开发服务器
4. 浏览器访问 `http://localhost:3000/admin/login` — 显示 ProForm 登录页
5. 输入 admin@example.com / password123 登录 → 跳转到 admin dashboard
6. Dashboard 显示 ProLayout 侧边栏 + 当前用户名
7. 点击退出 → 回到登录页
8. 未登录访问 `/admin` → 重定向到登录页
9. TypeScript 编译通过：`npm run check`
