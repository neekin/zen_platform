# Zen Platform 增强脚手架系统 - 深度规划

> 创建时间：2026-07-02
> 状态：待审批

---

## 概述

将现有脚手架从「简单 CRUD 生成器」升级为「产品形态定义系统」，通过 Model DSL 驱动前端页面自动生成，支持关联、多种字段组件、多种产品形态。

### 核心目标
1. **一条命令生成完整产品形态** - 包含关联、校验、组件选择
2. **Model DSL** - 声明式定义模型行为和展示方式
3. **关联自动处理** - belongs_to 自动变成下拉选择，has_many 自动变成子表格
4. **多种产品形态** - CRUD、看板、日历、仪表盘等

## Phase 1: Model DSL 设计

### 1.1 DSL 语法定义

```ruby
# app/models/article.rb
class Article < ApplicationRecord
  include Zen::Model DSL

  # 基础字段
  field :title, :string, required: true, placeholder: "请输入标题"
  field :body, :rich_text, required: true
  field :status, :enum, values: %w[draft published archived], default: "draft"
  field :priority, :integer, min: 0, max: 5, default: 0
  field :cover, :image, accept: %w[image/jpeg image/png], max_size: 10.megabytes
  field :attachments, :file, multiple: true, max_count: 10
  field :published_at, :datetime
  field :is_featured, :boolean, default: false
  field :price, :decimal, precision: 10, scale: 2
  field :tags, :tags, suggestions: %w[新闻 公告 教程]
  field :category_id, :reference, target: Category

  # 关联
  belongs_to :category
  has_many :comments
  has_many :article_tags
  has_many :tags, through: :article_tags

  # 展示配置
  display do
    list do
      column :title, link: true
      column :status, badge: true
      column :category, display: :name
      column :created_at, format: :relative_time
      column :cover, thumbnail: true
      column :actions, width: 200
    end

    form do
      section "基本信息" do
        field :title
        field :status, as: :radio
        field :category_id, as: :select
        field :priority, as: :slider
      end
      section "内容" do
        field :body, as: :rich_text
        field :cover, as: :image_upload
      end
      section "高级" do
        field :tags, as: :tags
        field :published_at, as: :datetime_picker
        field :is_featured, as: :switch
      end
    end

    detail do
      section "基本信息" do
        field :title, as: :heading
        field :status, as: :badge
        field :category
      end
      section "内容" do
        field :body, as: :rich_text_viewer
        field :cover, as: :image_preview
      end
      has_many :comments, display: :list
    end

    # 产品形态
    product :kanban, group_by: :status
    product :calendar, date_field: :published_at
  end

  # 校验
  validates :title, presence: true, length: { maximum: 255 }
  validates :status, inclusion: { in: %w[draft published archived] }
end
```

### 1.2 支持的字段类型

| 类型 | 表单组件 | 列表展示 | 详情展示 |
|------|----------|----------|----------|
| `:string` | Input | 文本 | 文本 |
| `:text` | TextArea | 截断 | 全文 |
| `:rich_text` | RichTextEditor | 纯文本摘要 | RichTextViewer |
| `:integer` | InputNumber | 数字 | 数字 |
| `:decimal` | InputNumber | 数字 | 数字 |
| `:boolean` | Switch | 标签 | 标签 |
| `:date` | DatePicker | 日期 | 日期 |
| `:datetime` | DatePicker(showTime) | 相对时间 | 完整时间 |
| `:enum` | Select/Radio | Badge | Badge |
| `:image` | ImageUpload | 缩略图 | 图片预览 |
| `:file` | FileUpload | 文件列表 | 文件列表 |
| `:tags` | TagInput | Tag 列表 | Tag 列表 |
| `:json` | CodeEditor | JSON 预览 | JSON 查看器 |
| `:reference` | Select(关联查询) | 关联显示名 | 关联详情 |
| `:color` | ColorPicker | 色块 | 色块 |
| `:url` | Input | 链接 | 链接 |
| `:email` | Input | 链接 | 链接 |

### 1.3 关联处理

```ruby
# belongs_to → 自动生成下拉选择
belongs_to :category
# 表单: <Select> 加载 Category.all
# 列表: 显示 category.name
# 详情: 显示 category 详情链接

# has_many → 自动生成子表格
has_many :comments
# 详情页: 显示 comments 列表
# 支持内联编辑

# has_many through → 自动生成多选
has_many :tags, through: :article_tags
# 表单: <Select mode="multiple">
# 列表: Tag 标签列表
# 详情: Tag 标签列表
```

## Phase 2: 增强生成器

### 2.1 新命令语法

```bash
# 基础用法（向后兼容）
rails generate zen:admin Article title:string body:text

# 使用 DSL（推荐）
rails generate zen:admin Article --dsl

# 指定产品形态
rails generate zen:admin Article --product=kanban
rails generate zen:admin Article --product=calendar

# 组合使用
rails generate zen:admin Article title:string status:enum[draft,published] \
  --rich-text=body --image=cover --belongs-to=category --has-many=comments
```

**文件**: `lib/generators/zen/admin_generator.rb`

- `--product` - 产品形态 (crud|kanban|calendar|dashboard)

## Phase 4: Controller 增强

### 4.1 关联数据加载

```ruby
# app/controllers/admin/articles_controller.rb
module Admin
  class ArticlesController < AdminController
    before_action :require_login
    before_action :set_article, only: [:show, :update, :destroy]

    def index
      @articles = Article
        .includes(:category, :tags)  # 预加载关联
        .order(created_at: :desc)
        .page(params[:page])

      render inertia: "admin/articles/Index",
        props: {
          articles: @articles.as_json(
            include: {
              category: { only: [:id, :name] },
              tags: { only: [:id, :name] }
            }
          ),
          # 关联数据（用于下拉选择）
          categories: Category.all.as_json(only: [:id, :name]),
          tags: Tag.all.as_json(only: [:id, :name]),
          # 分页
          pagination: {
            current: @articles.current_page,
            total: @articles.total_count,
            page_size: @articles.limit_value
          }
        }
    end

    def create
      @article = Article.new(article_params)
      if @article.save
        redirect_to admin_articles_path, notice: "创建成功"
      else
        redirect_to admin_articles_path, alert: @article.errors.full_messages.join(", ")
      end
    end

    private

    def article_params
      params.require(:article).permit(
        :title, :body, :status, :category_id, :cover,
        tag_ids: [], attachments: []
      )
    end
  end
end
```

### 4.2 关联 API 端点

```ruby
# 用于远程搜索关联数据
# GET /admin/articles/reference/category?q=关键字
def reference
  model = params[:model].classify.constantize
  query = params[:q]
  
  records = model.all
  records = records.where("name LIKE ?", "%#{query}%") if query.present?
  
  render json: {
    data: records.limit(20).as_json(only: [:id, :name])
  }
end
```

## 风险与缓解

| 风险 | 缓解措施 |
|------|----------|
| DSL 复杂度高 | 渐进式实现，先支持核心功能 |
| 关联查询性能 | 使用 `includes` 预加载，分页 |
| 组件包大小 | 按需加载，动态 import |
| 向后兼容 | 保持现有 `--modal/--page` 选项 |
