# API 错误响应格式

## 统一格式

所有 API 错误响应遵循以下格式：

```json
{
  "code": 1,
  "message": "人类可读的错误描述"
}
```

## 错误码

| HTTP Status | code | 含义 |
|-------------|------|------|
| 400 | 1 | 请求参数错误 |
| 401 | 1 | 未认证 |
| 403 | 1 | 无权限 |
| 404 | 1 | 资源不存在 |
| 422 | 1 | 验证失败 |
| 429 | 1 | 请求频率超限 |
| 500 | 1 | 服务器内部错误 |

## 示例

### 未认证 (401)
```json
{
  "code": 1,
  "message": "未授权"
}
```

### 验证失败 (422)
```json
{
  "code": 1,
  "message": "邮箱已被使用, 用户名不能为空"
}
```

### 限流 (429)
```json
{
  "error": "Rate limit exceeded. Retry later."
}
```

## 成功响应

```json
{
  "code": 0,
  "message": "成功",
  "data": { ... }
}
```
