# 任务管理流程

## 1. 任务拆分原则
- 每个任务只做**一件事**
- 任务完成时间不超过 **30 分钟**
- 任务描述必须包含**可验证的标准**

## 2. 任务格式
```
任务标题：[动词] + [具体目标]
验收标准：
- [ ] 代码变更：git diff 确认
- [ ] 测试通过：bundle exec rspec xxx
- [ ] 功能验证：具体操作步骤
```

## 3. 完成流程
```
创建任务 → 工程师执行 → 提交代码 → 我验证 → 确认完成
                                    ↓
                              不通过 → 退回重新执行
```

## 4. 验证命令模板
```bash
# 代码变更验证
git log --oneline -1
git diff HEAD~1 --stat

# 测试验证
bundle exec rspec [相关测试文件]

# 功能验证
# 具体操作步骤
```

## 5. 禁止行为
- ❌ 任务包含多个独立功能
- ❌ 标记 done 但没有 git commit
- ❌ 没有运行测试就提交
- ❌ 没有验证功能是否正常
