工作流引擎 — API 接口文档 ​

概述 ​

所有工作流相关的 API 都位于 /admin/workflow/ 命名空间下，需要管理员身份认证。 前端通过 Inertia.js 与后端交互，部分接口返回 JSON。

流程定义 (Definitions) ​

列表页 ​

GET /admin/workflow/definitions

页面组件: admin/workflows/Index

Props:

  • definitions — 流程定义列表，每项包含:
    • id, name, key, version, description, active, created_at
    • instance_count — 关联的实例数量
  • 分页和筛选参数

新建页 ​

GET /admin/workflow/definitions/new

页面组件: admin/workflows/Designer

Props:

  • workflow — null（新建模式）

设计器页 ​

GET /admin/workflow/definitions/:id/designer

页面组件: admin/workflows/Designer

Props:

  • workflow — 流程定义详情（包含 bpmn_definition JSON）

创建流程 ​

POST /admin/workflow/definitions

参数 (workflow_definition):

  • name (string, 必填) — 流程名称
  • key (string, 必填) — 流程标识（英文，唯一）
  • description (string, 可选) — 流程描述
  • bpmn_definition (json, 必填) — BPMN 定义 JSON
  • form_config (json, 可选) — 表单配置
  • active (boolean, 可选) — 是否启用，默认 true

成功响应 (200):

json
{ "id": 1, "notice": "创建成功" }

失败响应 (422):

json
{ "errors": ["名称不能为空", "标识已被使用"] }

更新流程 ​

PATCH /admin/workflow/definitions/:id

参数: 同创建，所有字段可选

成功响应 (200):

json
{ "notice": "保存成功" }

启用/禁用 ​

PATCH /admin/workflow/definitions/:id/toggle

切换流程的 active 状态。重定向回列表页。

删除流程 ​

DELETE /admin/workflow/definitions/:id

限制: 如果该流程下存在实例，删除将被拒绝。

流程实例 (Instances) ​

实例列表 ​

GET /admin/workflow/instances

页面组件: admin/workflows/Instances

查询参数:

  • status (string, 可选) — 按状态筛选: pending, running, completed, cancelled, failed
  • q (string, 可选) — 搜索关键词（匹配流程名称或标识）
  • page (integer, 可选) — 页码，默认 1
  • per_page (integer, 可选) — 每页数量，默认 20

Props:

  • instances — 实例列表，每项包含:
    • id, status, definition_name, definition_key
    • initiator_name — 发起人名称
    • variables — 流程变量
    • current_nodes — 当前活跃节点
    • started_at, completed_at, created_at
    • task_count — 关联任务数
  • pagination — 分页信息 { page, per_page, total }
  • filters — 当前筛选条件

实例详情 ​

GET /admin/workflow/instances/:id

页面组件: admin/workflows/Show

Props:

  • instance — 实例详情（同列表项字段）
  • tasks — 任务列表，每项包含:
    • id, node_id, node_name, task_type, status
    • assignee_name — 处理人名称
    • comment, form_data, due_date, completed_at, created_at
  • histories — 历史记录列表，每项包含:
    • id, node_id, node_name, node_type, action
    • operator_name — 操作人名称
    • comment, snapshot, executed_at
  • bpmn_definition — BPMN JSON 定义（用于渲染流程图）

终止流程 ​

POST /admin/workflow/instances/:id/cancel

参数:

  • reason (string, 可选) — 终止原因

限制: 已完成的流程无法终止。

任务管理 (Tasks) ​

任务列表 ​

GET /admin/workflow/tasks

页面组件: admin/workflow/tasks/Index

查询参数:

  • tab (string) — 视图切换:
    • pending（默认）— 待我处理
    • completed — 已完成
    • initiated — 我发起的
  • q (string, 可选) — 搜索关键词
  • page, per_page — 分页

Props:

  • tasks — 任务列表，每项包含:
    • id, node_id, node_name, task_type, status
    • assignee_name, comment, due_date, completed_at, created_at
    • instance_id, definition_name, initiator_name, instance_status
  • pagination — 分页信息
  • tab — 当前标签
  • filters — 筛选条件

完成任务 ​

POST /admin/workflow/tasks/:id/complete

参数:

  • comment (string, 可选) — 处理意见
  • form_data (json, 可选) — 表单数据

完成任务后引擎会自动推进流程到下一个节点。

驳回任务 ​

POST /admin/workflow/tasks/:id/reject

参数:

  • comment (string, 必填) — 驳回原因

转办任务 ​

POST /admin/workflow/tasks/:id/transfer

参数:

  • to_user_id (integer, 必填) — 转办目标用户 ID
  • comment (string, 可选) — 转办说明

催办任务 ​

POST /admin/workflow/tasks/:id/remind

向当前处理人发送催办通知。

审批服务 (Approval Service) ​

审批服务通过 Ruby 代码调用，不直接暴露 HTTP API。在控制器中使用方式：

发起审批 ​

ruby
Workflow::ApprovalService.submit(
  definition_key: "data_approval",
  initiator: current_user,
  title: "修改用户数据",
  approvers: [manager, director],
  form_data: { field: "email", old_value: "a@b.com", new_value: "c@d.com" }
)

审批通过 ​

ruby
Workflow::ApprovalService.approve(
  task: approval_task,
  user: current_user,
  comment: "同意"
)

审批驳回 ​

ruby
Workflow::ApprovalService.reject(
  task: approval_task,
  user: current_user,
  comment: "信息不完整，请补充"
)

撤回审批 ​

ruby
Workflow::ApprovalService.withdraw(
  instance: workflow_instance,
  user: current_user
)

查询接口 ​

ruby
# 待我审批
Workflow::ApprovalService.pending_approvals(user: current_user)

# 我发起的审批
Workflow::ApprovalService.my_submissions(user: current_user)

# 审批历史
Workflow::ApprovalService.approval_history(instance: workflow_instance)

任务服务 (Task Service) ​

ruby
# 分配任务
Workflow::TaskService.assign(task: task, user: user)

# 完成任务
Workflow::TaskService.complete(task: task, user: user, comment: "已审核")

# 驳回任务
Workflow::TaskService.reject(task: task, user: user, comment: "信息不完整")

# 转办任务
Workflow::TaskService.transfer(task: task, from_user: user, to_user: new_user)

# 催办
Workflow::TaskService.remind(task: task, sender: user)

# 查询
Workflow::TaskService.pending_tasks(user: user)
Workflow::TaskService.completed_tasks(user: user)
Workflow::TaskService.initiated_tasks(user: user)

引擎 API (Engine) ​

ruby
# 初始化引擎
definition = WorkflowDefinition.active.by_key("leave_request").first
engine = Workflow::Engine.new(definition: definition)

# 启动流程
instance = engine.start!(
  initiator: current_user,
  variables: { leave_type: "annual", days: 3 }
)

# 推进流程（任务完成后）
task = WorkflowTask.find(task_id)
engine = Workflow::Engine.new(instance: task.workflow_instance)
engine.advance!(
  task: task,
  result: { approved: true, comment: "同意" }
)

# 获取当前任务
engine.current_tasks

# 管理变量
engine.variables
engine.update_variables(amount: 5000)
