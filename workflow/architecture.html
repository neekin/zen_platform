工作流引擎 — 架构说明 ​

概述 ​

Zen Platform 工作流引擎是一个基于 BPMN（业务流程建模与标注）的轻量级流程引擎，支持可视化流程设计、 自动任务分配、审批流转和流程监控。引擎采用 Ruby on Rails 实现，前端使用 React + Ant Design + X6 提供可视化流程设计器。

核心组件 ​

1. Parser（BPMN 解析器） ​

位置: lib/workflow/parser.rb

负责解析存储在 workflow_definitions.bpmn_definition 中的 JSON 结构。解析后的数据包含节点（nodes）和 流（flows），并通过结构校验确保定义的完整性。

支持的节点类型:

  • start_event — 开始事件，流程入口
  • end_event — 结束事件，流程出口
  • user_task — 用户任务，需要人工处理
  • approval_task — 审批任务，需要审批决策
  • service_task — 服务任务（预留）
  • exclusive_gateway — 排他网关，条件分支（只走一条路径）
  • parallel_gateway — 并行网关，同时执行多条路径
  • inclusive_gateway — 包含网关（预留）

数据格式:

json
{
  "nodes": [
    { "id": "start", "type": "start_event", "name": "开始" },
    { "id": "task1", "type": "approval_task", "name": "经理审批",
      "candidate_users": ["1", "2"], "assignee": "${initiator}" },
    { "id": "end", "type": "end_event", "name": "结束" }
  ],
  "flows": [
    { "id": "f1", "source": "start", "target": "task1" },
    { "id": "f2", "source": "task1", "target": "end",
      "condition": "approved == true" }
  ]
}

2. Engine（流程引擎） ​

位置: lib/workflow/engine.rb

核心引擎类，负责启动流程实例、推进流程执行和管理流程变量。

主要方法:

  • start!(initiator:, variables:) — 启动新的工作流实例
  • advance!(task:, result:) — 完成任务后推进流程
  • current_tasks — 获取当前待办任务
  • variables / update_variables — 管理流程变量

执行流程:

  1. 解析 BPMN 定义
  2. 创建 WorkflowInstance
  3. 从 start_event 开始，顺序执行节点
  4. 遇到 user_task / approval_task 时创建任务并暂停
  5. 任务完成后调用 advance! 继续执行
  6. 遇到 gateway 时评估条件或并行分支
  7. 到达 end_event 时完成流程

3. Executor（执行器） ​

位置: lib/workflow/executor.rb

负责具体的节点执行逻辑，与 Engine 配合完成流程推进。提供任务自动分配、条件评估、历史记录等功能。

4. ApprovalService（审批服务） ​

位置: app/services/workflow/approval_service.rb

提供审批流程的高级业务逻辑，包括：

  • submit — 发起审批（创建实例 + 分配首个审批人）
  • approve — 审批通过（推进到下一个审批人或完成）
  • reject — 审批驳回（终止流程并通知发起人）
  • withdraw — 撤回审批（仅发起人可操作）
  • pending_approvals — 查询待我审批
  • my_submissions — 查询我发起的审批
  • approval_history — 查询审批历史

5. TaskService（任务服务） ​

位置: app/services/workflow/task_service.rb

提供任务管理的业务逻辑：

  • assign — 分配任务给用户
  • assign_to_role — 分配任务给角色
  • complete — 完成任务
  • reject — 驳回任务
  • transfer — 转办任务
  • remind — 催办任务
  • pending_tasks / completed_tasks / initiated_tasks — 查询任务

条件表达式 ​

排他网关的条件支持以下格式：

variable operator value

支持的操作符: ==, !=, >, <, >=, <=

示例:

  • approved == true — 审批通过
  • amount > 1000 — 金额大于 1000
  • status != "rejected" — 状态不是已拒绝

任务自动分配 ​

支持通过 assignee 属性自动分配任务：

  • 变量引用: ${initiator} — 从流程变量中获取用户 ID
  • 直接指定: 用户 ID 字符串

通知集成 ​

引擎通过 NotificationService 发送通知：

  • approval_pending — 有待审批任务
  • approval_approved — 审批通过
  • approval_rejected — 审批驳回
  • workflow_task_assigned — 任务已分配
  • workflow_task_completed — 任务已完成
  • workflow_task_rejected — 任务被驳回
  • workflow_task_transferred — 任务已转办
  • workflow_task_reminder — 任务催办提醒
