工作流引擎 — Demo 流程 ​

Demo 1: 请假审批流程 ​

场景描述 ​

员工提交请假申请，直属经理审批。如果请假天数超过 3 天，还需要部门总监审批。 经理或总监驳回则流程终止，通知发起人。

流程图 ​

[开始] → [填写请假申请(用户任务)] → [经理审批(审批任务)] → 排他网关
                                                          ├─ approved == true → 排他网关(天数判断)
                                                          │                      ├─ days > 3 → [总监审批] → 排他网关
                                                          │                      │                              ├─ approved → [结束(通过)]
                                                          │                      │                              └─ default → [结束(驳回)]
                                                          │                      └─ days <= 3 → [结束(通过)]
                                                          └─ default → [结束(驳回)]

BPMN 定义 ​

json
{
  "nodes": [
    { "id": "start", "type": "start_event", "name": "开始" },
    {
      "id": "fill_form",
      "type": "user_task",
      "name": "填写请假申请",
      "assignee": "${initiator}",
      "form_key": "leave_request_form"
    },
    {
      "id": "manager_approval",
      "type": "approval_task",

    },
    {
      "id": "gateway_approved",

      "default_flow": "f_rejected_1"
    },

      "id": "gateway_days",

    },
    {
      "id": "director_approval",
      "type": "approval_task",
      "name": "总监审批",
      "candidate_roles": ["director"],

    },
    {
      "id": "gateway_director",
      "type": "exclusive_gateway",

  ]

使用代码 ​

ruby
# 发起请假申请
definition = WorkflowDefinition.active.by_key("leave_request").first
engine = Workflow::Engine.new(definition: definition)

instance = engine.start!(
  initiator: current_user,
  variables: { leave_type: "annual", days: 5, reason: "年假" }
)

# 经理审批通过
manager_task = instance.workflow_tasks.find_by(node_id: "manager_approval")
engine = Workflow::Engine.new(instance: instance)
engine.advance!(task: manager_task, result: { approved: true, comment: "同意" })

# 总监审批通过
director_task = instance.workflow_tasks.find_by(node_id: "director_approval")
engine.advance!(task: director_task, result: { approved: true, comment: "批准" })

Demo 2: 数据审批流程 ​

场景描述 ​

运营人员需要修改线上数据（如用户信息、配置参数），提交数据变更申请。数据管理员审核 变更内容的准确性和影响范围，审批通过后自动执行变更，驳回则通知发起人。

流程图 ​

[开始] → [提交数据变更申请(用户任务)] → [数据管理员审批(审批任务)] → 排他网关
                                                                  ├─ approved == true → [结束(执行变更)]
                                                                  └─ default → [结束(驳回)]

BPMN 定义 ​

json
{
  "nodes": [
    { "id": "start", "type": "start_event", "name": "开始" },
    {
      "id": "submit_request",
      "type": "user_task",
      "name": "提交数据变更申请",
      "assignee": "${initiator}",
      "form_key": "data_change_form"
    },
    {
      "id": "data_admin_approval",
      "type": "approval_task",
      "name": "数据管理员审批",
      "candidate_roles": ["data_admin"],
      "due_duration": "P1D"
    },
    {
      "id": "gateway_result",
      "type": "exclusive_gateway",
      "outgoing": ["f_approved", "f_rejected"],
      "default_flow": "f_rejected"
    },
    { "id": "end_approved", "type": "end_event", "name": "执行变更" },
    { "id": "end_rejected", "type": "end_event", "name": "驳回" }
  ],
  "flows": [
    { "id": "f1", "source": "start", "target": "submit_request" },
    { "id": "f2", "source": "submit_request", "target": "data_admin_approval" },
    { "id": "f3", "source": "data_admin_approval", "target": "gateway_result" },
    { "id": "f_approved", "source": "gateway_result", "target": "end_approved", "condition": "approved == true" },
    { "id": "f_rejected", "source": "gateway_result", "target": "end_rejected" }
  ]
}

使用代码 ​

ruby
# 方式一：使用 ApprovalService（推荐）
instance = Workflow::ApprovalService.submit(
  definition_key: "data_approval",
  initiator: current_user,
  title: "修改用户邮箱",
  approvers: [data_admin],
  form_data: {
    table: "users",
    record_id: 42,
    field: "email",
    old_value: "old@example.com",
    new_value: "new@example.com"
  }
)

# 数据管理员审批
pending_task = Workflow::ApprovalService.pending_approvals(user: data_admin).first
Workflow::ApprovalService.approve(
  task: pending_task,
  user: data_admin,
  comment: "已核对，变更正确"
)

# 或者驳回
Workflow::ApprovalService.reject(
  task: pending_task,
  user: data_admin,
  comment: "新邮箱格式不正确，请修正后重新提交"
)
ruby
# 方式二：使用 Engine（更灵活）
definition = WorkflowDefinition.active.by_key("data_approval").first
engine = Workflow::Engine.new(definition: definition)

instance = engine.start!(
  initiator: current_user,
  variables: {
    table: "users",
    record_id: 42,
    field: "email",
    old_value: "old@example.com",
    new_value: "new@example.com"
  }
)

# 提交申请
submit_task = instance.workflow_tasks.find_by(node_id: "submit_request")
engine.advance!(task: submit_task, result: {
  comment: "请审核邮箱变更",
  form_data: { confirmed: true }
})

# 审批通过
approval_task = instance.workflow_tasks.find_by(node_id: "data_admin_approval")
engine.advance!(task: approval_task, result: {
  approved: true,
  comment: "审核通过"
})

Demo 3: 使用 ApprovalService 快速发起审批 ​

对于简单的审批场景，可以直接使用 ApprovalService，无需设计完整的 BPMN 流程：

ruby
# 发起一个 3 级串行审批
instance = Workflow::ApprovalService.submit(
  definition_key: "simple_approval",
  initiator: current_user,
  title: "年度预算审批",
  approvers: [manager, director, cfo],
  variables: { budget_amount: 100000 }
)

# 查看审批进度
instance.workflow_histories.chronological.each do |h|
  puts "#{h.executed_at} - #{h.action} - #{h.comment}"
end
