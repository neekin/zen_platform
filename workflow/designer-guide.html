流程设计器 — 使用指南 ​

概述 ​

流程设计器是 Zen Platform 工作流引擎的可视化编辑工具，基于 AntV X6 图引擎构建。用户可以通过 拖拽节点、连线和配置属性来设计业务流程，无需编写代码。

访问入口 ​

  • 新建流程: 导航至 /admin/workflow/definitions/new
  • 编辑流程: 在流程列表页点击某个流程的"设计"按钮，跳转至 /admin/workflow/definitions/:id/designer

节点类型 ​

1. 开始事件 (Start Event) ​

  • 图标: 绿色圆圈
  • 作用: 流程的入口点，每个流程必须有且只有一个开始事件
  • 属性: 无特殊配置
  • 规则: 只能有出边，不能有入边

2. 结束事件 (End Event) ​

  • 图标: 红色圆圈
  • 作用: 流程的终点，一个流程可以有多个结束事件
  • 属性: 无特殊配置
  • 规则: 只能有入边，不能有出边

3. 用户任务 (User Task) ​

  • 图标: 蓝色矩形（带人形标记）
  • 作用: 需要人工处理的任务节点
  • 属性:
    • name — 任务名称（必填）
    • candidate_users — 候选用户 ID 列表
    • candidate_roles — 候选角色名称列表
    • assignee — 自动分配规则，支持 ${variable} 语法
    • due_duration — 截止时间（ISO 8601 格式，如 P1D 表示 1 天）
    • form_key — 关联的表单标识

4. 审批任务 (Approval Task) ​

  • 图标: 橙色矩形（带勾选标记）
  • 作用: 需要审批决策的任务节点，审批结果会影响流程走向
  • 属性:
    • name — 审批名称（必填）
    • candidate_users — 候选审批人列表
    • candidate_roles — 候选审批角色
    • assignee — 自动分配规则
    • due_duration — 审批截止时间
  • 特殊行为: 审批通过/驳回后会设置流程变量 approved = true/false

5. 排他网关 (Exclusive Gateway) ​

  • 图标: 菱形（带 × 标记）
  • 作用: 条件分支，根据条件表达式选择唯一的执行路径
  • 属性:
    • outgoing — 出边列表
    • default_flow — 默认流（当所有条件都不满足时走这条路径）
  • 规则: 每条出边可以设置 condition，引擎按顺序评估，第一个满足条件的路径被选中

6. 并行网关 (Parallel Gateway) ​

  • 图标: 菱形（带 + 标记）
  • 作用: 同时执行所有分支路径
  • 属性: 无特殊配置
  • 行为: 所有出边同时执行，所有入边都到达后才继续

操作指南 ​

添加节点 ​

  1. 从左侧面板拖拽节点类型到画布区域
  2. 节点会自动添加到画布中
  3. 点击节点在右侧属性面板中配置属性

连接节点 ​

  1. 将鼠标悬停在节点上，出现连接点
  2. 从源节点的连接点拖拽到目标节点
  3. 松开鼠标完成连线
  4. 排他网关的出边需要在属性面板中设置条件

配置属性 ​

  1. 点击画布中的节点或连线
  2. 右侧属性面板显示可编辑的属性
  3. 修改属性后自动保存到画布

删除元素 ​

  • 选中节点或连线后按 Delete 键
  • 或右键选择"删除"

撤销 / 重做 ​

  • 快捷键 Ctrl+Z（撤销）/ Ctrl+Shift+Z（重做）
  • 或点击工具栏中的撤销/重做按钮

保存流程 ​

  1. 点击工具栏的"保存"按钮
  2. 首次保存需要填写：
    • 流程名称
    • 流程标识（英文 key，创建后不可修改）
    • 流程描述（可选）
  3. 保存后会自动将画布内容转换为 BPMN JSON 存储到 workflow_definitions.bpmn_definition

流程校验 ​

保存时引擎会自动校验：

  • ✅ 必须有开始事件
  • ✅ 必须有结束事件
  • ✅ 所有流的源和目标节点必须存在
  • ✅ 网关必须有出边
  • ✅ 非开始/结束节点不能是孤立节点（必须有入边和出边）

校验失败时会显示错误信息，修复后才能保存。

常用流程模式 ​

简单审批流 ​

开始 → 审批任务 → 排他网关
                    ├─ approved == true → 结束
                    └─ (默认) → 驳回通知 → 结束

串行多级审批 ​

开始 → 经理审批 → 排他网关
                  ├─ approved → 总监审批 → 排他网关
                  │                          ├─ approved → 结束
                  │                          └─ default → 退回经理
                  └─ default → 驳回通知 → 结束

并行会签 ​

开始 → 并行网关 → 审批人A → ┐
              → 审批人B → ├→ 并行网关（汇聚）→ 结束
              → 审批人C → ┘
