工作流 (Workflow) ​

Zen Platform 内置可视化工作流引擎，支持条件分支、并行执行、定时触发，适合业务流程自动化。

付费功能

工作流是付费功能，定价 ¥200。联系作者获取授权。

快速开始 ​

1. 创建工作流 ​

访问 Admin → 工作流，点击"新建工作流"：

yaml
名称: 文章审核流程
触发方式: 手动触发
节点:
  - 开始
  - 条件判断: status == "pending"
  - 发送通知
  - 结束

2. 设计流程 ​

使用可视化设计器拖拽节点：

  • 触发节点 — 手动触发、定时触发、Webhook
  • 动作节点 — 创建记录、更新记录、发送通知、调用 API
  • 条件节点 — IF/ELSE 分支判断
  • 并行节点 — 同时执行多个分支

3. 执行工作流 ​

ruby
# 手动触发
Workflow.find(1).execute(params: { article_id: 42 })

# 定时触发（自动执行）
# 在设计器中设置 cron 表达式

节点类型 ​

触发节点 ​

类型说明示例
手动用户手动触发审核按钮
定时Cron 表达式触发每天 9:00
WebhookHTTP 请求触发外部系统回调
模型事件记录创建/更新/删除新订单创建

动作节点 ​

类型说明参数
创建记录创建数据库记录模型、字段值
更新记录更新数据库记录模型、ID、字段值
删除记录删除数据库记录模型、ID
发送通知发送系统通知接收人、内容
调用 APIHTTP 请求URL、方法、参数
延时等待指定时间秒数
自定义执行 Ruby 代码代码块

条件节点 ​

yaml
条件:
  - if: record.status == "pending"
    then: 发送通知
  - elif: record.status == "approved"
    then: 更新状态
  - else:
    then: 记录日志

支持的操作符：

  • ==, !=, >, <, >=, <=
  • include?, present?, blank?
  • match? (正则)

并行节点 ​

yaml
并行:
  - 分支1: 发送邮件通知
  - 分支2: 更新统计表
  - 分支3: 调用外部 API
等待: 全部完成

设计器使用 ​

打开设计器 ​

访问 /admin/workflows/:id/designer

操作说明 ​

  1. 添加节点 — 从左侧拖拽到画布
  2. 连接节点 — 拖拽节点底部连接点到下一个节点
  3. 配置节点 — 点击节点，在右侧面板配置参数
  4. 删除节点 — 选中节点，按 Delete 键

快捷键 ​

快捷键功能
Ctrl+S保存
Ctrl+Z撤销
Ctrl+Y重做
Delete删除选中节点
空格+拖拽平移画布
滚轮缩放画布

示例：文章审核流程 ​

yaml
工作流: 文章审核
触发: 模型事件 - Article 创建

节点:
  1. 开始
  2. 条件判断:
     if: record.status == "pending"
  3. 发送通知:
     接收人: 角色(editor)
     内容: "新文章待审核: {{record.title}}"
  4. 等待:
     等待人工审核（通过 Webhook 回调）
  5. 条件判断:
     if: params.approved == true
     then: 更新状态为 published
     else: 更新状态为 rejected
  6. 发送通知:
     接收人: record.user
     内容: "文章审核结果: {{record.status}}"
  7. 结束

示例：定时数据统计 ​

yaml
工作流: 每日统计
触发: 定时 - 每天 09:00

节点:
  1. 开始
  2. 创建记录:
     模型: DailyStat
     字段:
       date: today
       user_count: User.count
       order_count: Order.where("created_at > ?", 1.day.ago).count
  3. 条件判断:
     if: record.order_count > 100
     then: 发送通知 "今日订单突破100！"
  4. 结束

示例：多级审批 ​

yaml
工作流: 采购审批
触发: 手动触发

节点:
  1. 开始
  2. 发送通知:
     接收人: 角色(manager)
     内容: "采购申请待审批"
  3. 等待: Webhook 回调
  4. 条件判断:
     if: params.approved && params.amount < 10000
     then: 自动通过
  5. 条件判断:
     if: params.amount >= 10000
     then: 发送通知给角色(director)
          等待: Webhook 回调
  6. 更新记录:
     status: params.final_status
  7. 发送通知:
     接收人: record.applicant
     内容: "采购申请结果: {{record.status}}"
  8. 结束

API 接口 ​

执行工作流 ​

POST /api/v1/workflows/:id/execute
Authorization: Bearer <token>
Content-Type: application/json

{
  "params": {
    "article_id": 42,
    "approved": true
  }
}

查询执行记录 ​

GET /api/v1/workflow_executions?workflow_id=1
Authorization: Bearer <token>

常见问题 ​

工作流超时 ​

默认超时 30 分钟，可在工作流设置中调整。

如何调试 ​

  1. 查看执行日志：/admin/workflow_executions
  2. 使用"测试执行"模式
  3. 检查节点输出

性能考虑 ​

  • 避免在循环中执行耗时操作
  • 使用并行节点提高效率
  • 大批量数据使用延时节点分批处理
